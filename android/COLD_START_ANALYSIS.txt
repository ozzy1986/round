ROOT CAUSES (evidence-only)
---------------------------
1. JIT compilation on first frame
   - Log: "Compiler allocated 6110KB to compile void android.view.ViewRootImpl.performTraversals()"
   - performTraversals() is the measure/layout/draw pass. It is JIT-compiled when first used.
   - Impact: Large one-time cost on first launch (debug + no AOT/baseline profile).

2. First-frame composition cost (Davey 1579 ms)
   - One frame from start to completion = 1579 ms. QueueBufferDuration ≈ 565 ms (GPU).
   - Implies main-thread work for that frame ≈ 1000+ ms (Compose + Material 3 + NavHost + list).
   - Combined with JIT of performTraversals, the first draw is dominated by:
     - Compilation of view/layout code
     - Compose composition (lock verification warnings → unoptimized snapshot code)
     - Full tree measure/layout/draw

3. Compose snapshot running in slow path
   - Log: "SnapshotStateList ... failed lock verification and will run slower"
   - Debug dex / missing baseline profile → lock verification fails → slower runtime.

4. Two Choreographer skips (30 + 61 = 91 frames ≈ 1.5 s at 60 fps)
   - First 30: activity/window/surface setup (OEM + system).
   - Second 61: first Compose-driven frame (setContent → RaundTheme → NavHost → ProfileListScreen with 4 items).

5. OEM / device (TECNO)
   - TranWindowConfigurationImpl, TranChoreographerImpl, config_debug, FBI.
   - Not reducible in app code.

  Then rebuild release/debug; the generated file is merged into the app by the plugin.

- Three-stage deferred launch (MainActivity):
  1. installSplashScreen() with immediate dismiss (setKeepOnScreenCondition { false })
  2. Native FrameLayout placeholder (solid dark background) set via setContentView — draws
     as the first frame with zero Compose overhead
  3. Compose (RaundTheme + full UI) initialized via placeholder.post{} after first frame;
     reportFullyDrawn() called when ready

- Application.onCreate: zero main-thread work. No eager database init, no profileRepository
  access. All init (profileRepository lazy, preloadCache, requestSync) runs on Dispatchers.IO.
  requestSync() in a separate coroutine so it doesn't block repositoryReady.
