CORE PRINCIPLES (MANDATORY)
---------------------------
1. Focus on security, speed, stability, scalability.
2. DRY, KISS, SOLID.

1. WHAT IT IS
--------------
Interval timer for boxing, MMA, HIIT, running. Users define profiles: each profile has a name, an emoji, and an ordered list of rounds. Each round has a name (announced at start), duration in seconds, and an optional "warn 10 sec" (voice "X in 10 seconds" plus per-second tick in the last 10 seconds). One tap to start; app announces round names by voice and warnings. Works offline on the phone; syncs with the backend when online. Delivered as: backend API + Telegram bot (both run on the VPS), and an Android app that connects to the VPS server.


2. KEY FEATURES (from plan)
---------------------------
- Profiles: name, user-chosen emoji, list of rounds (order matters).
- Round: name (spoken at start), durationSeconds, warn10sec (voice "X in 10 sec" + tick each second).
- Quick entry: template "N x (work + rest)" generates N pairs (e.g. "3x5 1" = 3 x (5 min work + 1 min rest)); custom rounds (e.g. "5 run, 5 walk, 4 run, 15 walk").
- Timer screen: big countdown + current round emoji (profile emoji or round-based).
- i18n: EN + RU from start; strings in backend and Android resources.
- Offline-first: Android works without network; backend used for sync and Telegram.


3. ARCHITECTURE
---------------
- Server: API and Telegram bot run on the VPS (round.ozzy1986.com). The Android app and Telegram users connect to this server. The local machine is for development only (running backend/bot locally, emulator).
- Backend (this repo): Node.js, TypeScript, Fastify, PostgreSQL. REST API for profiles and rounds CRUD. Telegram bot (Telegraf) runs the same timer logic and sends TTS voice messages. Timer engine is a shared pure TS module (no I/O).
- Android app: in subfolder android/. Kotlin, Jetpack Compose, Room for local storage, Retrofit for sync with backend on VPS. Timer runs locally with same event contract; TTS and optional tick sounds.
- Shared timer contract: events emitted are round_start, tick (remainingSeconds), warn10, round_end, training_end. Backend uses the TS engine; Android reimplements the same logic in Kotlin.

6. DATA MODEL (PostgreSQL)
--------------------------
profiles:
  id (uuid, PK), name (text), emoji (text), user_id (text, nullable), created_at, updated_at (timestamptz)

rounds:
  id (uuid, PK), profile_id (uuid, FK -> profiles, CASCADE), name (text), duration_seconds (int), warn10sec (boolean), position (int)
  Index on profile_id; unique (profile_id, position).

Auth: not implemented yet (user_id nullable; API is anonymous).


7. API ENDPOINTS
---------------
  GET    /profiles              - list all profiles
  GET    /profiles/:id          - get profile with rounds (404 if not found)
  POST   /profiles              - create profile (body: name, emoji)
  PATCH  /profiles/:id          - update profile (body: name?, emoji?)
  DELETE /profiles/:id          - delete profile (cascades to rounds)

  GET    /profiles/:id/rounds   - list rounds for profile (ordered by position)
  POST   /profiles/:id/rounds   - create round (body: name, duration_seconds, warn10sec?, position)
  PATCH  /rounds/:roundId      - update round
  DELETE /rounds/:roundId      - delete round

All request/response bodies validated via Fastify JSON schema. UUIDs in path.


8. TIMER ENGINE (shared logic)
------------------------------
Input: TimerProfile { name, emoji, rounds: [{ name, durationSeconds, warn10sec }] }.
Output: stream of events (callback or advance() per second).

Events:
  round_start  - start of a round (roundIndex, round, totalRounds)
  tick         - every second (roundIndex, round, remainingSeconds, totalRounds)
  warn10       - when remainingSeconds === 10 and warn10sec (roundIndex, round, totalRounds)
  round_end    - end of round
  training_end - all rounds done

Pure logic; no I/O. Used by backend (Telegram) and by Android (Kotlin reimplementation).


9. TELEGRAM BOT
---------------
UI: Button-based (inline keyboards). No need to type commands; user taps buttons.

/start: Welcome message (i18n by user language_code) with two buttons: "Profiles", "Quick timer".
  - Profiles: Lists profiles from API as buttons (emoji + name); tap to start that profile's timer. "Back" returns to main menu.
  - Quick timer: Presets as buttons — e.g. 3×5+1, 5×3+1, "5 run, 5 walk". "Back" returns to main menu.
Commands (still supported): /profiles (same list with buttons), /timer <spec> (with no arg shows main menu; with arg starts timer).
<spec> for /timer: "3x5 1" (template), "5 run, 5 walk" (custom), or profile name/ID from API.

Flow: runs timer-engine in process; every second calls advance(). On round_start and warn10, synthesizes speech via Google TTS and sends voice message. On training_end sends text "Training finished." One running timer per user (stored in memory).

Env: TELEGRAM_BOT_TOKEN required. On VPS, API_URL is set so the bot talks to the local API (127.0.0.1:3001). Optional: GOOGLE_APPLICATION_CREDENTIALS for TTS.


10. ANDROID APP
---------------
Location: android/ (Gradle, Kotlin, Compose).

Screens: profile list, profile/round editor (name, emoji, rounds with name/duration/warn10 and label), timer (big countdown, current round name and emoji, TTS). One-tap start. Data in Room; sync with backend on VPS when online (Retrofit): sync from API when profile list is shown; profile and rounds create/update/delete push to API when online. Timer: same event contract; TTS for round name and "X in 10 seconds"; if profile has no rounds, message is shown and Start is disabled. i18n: values/strings.xml (EN), values-ru/strings.xml (RU).

Package: com.raund.app. API base URL in RaundApplication points to the VPS: http://round.ozzy1986.com:8080/ (for real device / phone). For Android emulator only use http://10.0.2.2:3001/.

Testing on phone: the app is usually already installed. Just launch it via ADB
(see ANDROID_TESTING.txt). Only rebuild the APK if android/ code was changed.
  - Rebuild (only when needed): from project root run "scripts\build-apk.ps1" (or "cd android && .\gradlew.bat assembleDebug"). The script copies the APK to android/Round-debug.apk.
  - Android Studio: open the android/ folder as project, then Build > Build Bundle(s) / APK(s) > Build APK(s). Output: android/app/build/outputs/apk/debug/app-debug.apk.
  - APK for transfer: android/Round-debug.apk (or app-debug.apk from the path above). Copy to the phone via USB, cloud, or email; open on the phone to install (enable "Install from unknown sources" if prompted). No app store required.
  - Build from PC when Google is blocked: gradle.properties has systemProp.http.proxyHost/Port and https equivalents pointing at the project's proxy (VPS Squid). Use proxy for Gradle only; the app on the phone needs no proxy and connects directly to the VPS API.

12. ENVIRONMENT & DEPLOYMENT
----------------------------
Production server (PRIMARY): the VPS at round.ozzy1986.com. The Android app and
Telegram bot connect here. The backend API + PostgreSQL + PM2 all run on the VPS.
This is the running system — no local setup needed to use or test the app.

Local machine (dev only — NOT required for running/testing the app):
  - Used only for: editing code, running Vitest, building APKs, then deploying to VPS.
  - Node.js, npm. PostgreSQL only if running DB/route tests locally (Vitest).
  - .env: PG_* or DATABASE_URL, PORT=3001, TELEGRAM_BOT_TOKEN (optional for bot), GOOGLE_APPLICATION_CREDENTIALS (optional for TTS).
  - npm run migrate:up, npm run dev (or npm run build && npm start).
  - Optional: Nginx or similar to proxy a hostname to 3001.

Git remotes (4-way sync):
  - origin  = GitHub  (https://github.com/ozzy1986/round.git)
  - gitlab  = GitLab  (https://gitlab.com/ozzy1986/round.git)
  - Workflow: commit locally -> push origin main -> push gitlab main -> deploy on VPS (see below). .env is not in the repo and is not overwritten on deploy.

VPS (production server):
  - Domain: round.ozzy1986.com (A record to VPS IP). Apache on 80; Nginx on 8080.
  - App dir: /var/www/round.ozzy1986.com (git clone from GitHub). .env in app root (PG_USER=round, PG_DATABASE=round, PORT=3001, HOST=127.0.0.1, TELEGRAM_BOT_TOKEN); .env persists across deploys.
  - Nginx: config in deploy/nginx-round.ozzy1986.com.conf (listen 8080, server_name round.ozzy1986.com, proxy_pass to 127.0.0.1:3001). On server, use /etc/nginx/sites-available/ copy or symlink; only one server block per server_name on 8080.
  - Deploy: on VPS run "cd /var/www/round.ozzy1986.com && bash scripts/deploy-vps.sh" (git pull, npm ci, migrate:up, build, pm2 restart round-api and round-bot). From local PC: scripts/deploy.ps1 (push both remotes then SSH and run deploy-vps.sh); requires SSH key from PC to VPS (ssh-copy-id root@VPS_IP).
  - PM2: round-api (HTTP server), round-bot (Telegram bot). PostgreSQL user and database: round.
  - For step-by-step sync procedure (for AI/next session), see SYNC.txt.
