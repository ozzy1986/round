1. WHAT IT IS
--------------
Interval timer for boxing, MMA, HIIT, running. Users define profiles: each profile has a name, an emoji, and an ordered list of rounds. Each round has a name (announced at start), duration in seconds, and an optional "warn 10 sec" (voice "X in 10 seconds" plus per-second tick in the last 10 seconds). One tap to start; app announces round names by voice and warnings. Works offline on the phone; syncs with the backend when online. Delivered as: backend API + Telegram bot (both run on the VPS), and an Android app that connects to the VPS server.


2. KEY FEATURES (from plan)
---------------------------
- Profiles: name, user-chosen emoji, list of rounds (order matters).
- Round: name (spoken at start), durationSeconds, warn10sec (voice "X in 10 sec" + tick each second).
- Quick entry: template "N x (work + rest)" generates N pairs (e.g. "3x5 1" = 3 x (5 min work + 1 min rest)); custom rounds (e.g. "5 run, 5 walk, 4 run, 15 walk").
- Timer screen: big countdown + current round emoji (profile emoji or round-based).
- i18n: EN + RU from start; strings in backend and Android resources.
- Offline-first: Android works without network; backend used for sync and Telegram.


3. ARCHITECTURE
---------------
- Server: API and Telegram bot run on the VPS (round.ozzy1986.com). The Android app and Telegram users connect to this server. The local machine is for development only (running backend/bot locally, emulator).
- Backend (this repo): Node.js, TypeScript, Fastify, PostgreSQL. REST API for profiles and rounds CRUD. Telegram bot (Telegraf) runs the same timer logic and sends TTS voice messages. Timer engine is a shared pure TS module (no I/O).
- Android app: in subfolder android/. Kotlin, Jetpack Compose, Room for local storage, Retrofit for sync with backend on VPS. Timer runs locally with same event contract; TTS and optional tick sounds.
- Shared timer contract: events emitted are round_start, tick (remainingSeconds), warn10, round_end, training_end. Backend uses the TS engine; Android reimplements the same logic in Kotlin.

6. DATA MODEL (PostgreSQL)
--------------------------
profiles:
  id (uuid, PK), name (text), emoji (text), user_id (text, nullable), created_at, updated_at (timestamptz)
  Index: (user_id, updated_at DESC) for list/get by user and delta sync.

rounds:
  id (uuid, PK), profile_id (uuid, FK -> profiles, CASCADE), name (text), duration_seconds (int), warn10sec (boolean), position (int)
  Index on profile_id; unique (profile_id, position). Trigger updates profile.updated_at when rounds change (for delta sync).

users:
  id (uuid, PK), is_anonymous (boolean), telegram_id (bigint, unique, nullable), created_at (timestamptz)

refresh_tokens:
  id (uuid, PK), token_hash (text), user_id (uuid, FK -> users, CASCADE), expires_at (timestamptz), revoked_at (timestamptz), created_at (timestamptz)

Auth: JWT-based. Access token 15m; refresh token 30 days, rotated on use. Anonymous registration (POST /auth/register) and Telegram auth (POST /auth/telegram with X-Bot-Secret) return token + refresh_token. POST /auth/refresh exchanges refresh_token for new token pair; POST /auth/logout revokes refresh_token. User isolation: all profile/round queries filter by user_id.


7. API ENDPOINTS
---------------
  GET    /profiles              - list profiles (query: limit?, cursor?). With include=rounds and optional updated_since= (ISO-8601): returns profiles with rounds in one response (batch + delta sync).
  GET    /profiles/:id          - get profile with rounds (404 if not found)
  POST   /profiles              - create profile (body: name, emoji)
  PATCH  /profiles/:id          - update profile (body: name?, emoji?)
  DELETE /profiles/:id          - delete profile (cascades to rounds)

  GET    /profiles/:id/rounds   - list rounds for profile (ordered by position)
  POST   /profiles/:id/rounds   - create round (body: name, duration_seconds, warn10sec?, position)
  PUT    /profiles/:id/rounds   - replace all rounds for profile (body: rounds: [{ name, duration_seconds, warn10sec?, position }]); atomic.
  PATCH  /rounds/:roundId      - update round
  DELETE /rounds/:roundId      - delete round

  POST   /auth/register        - create anonymous user; returns token, refresh_token, user_id
  POST   /auth/telegram        - (X-Bot-Secret) create/find user by telegram_id; returns token, refresh_token, user_id
  POST   /auth/refresh         - body: refresh_token; returns new token, refresh_token, user_id (rotation)
  POST   /auth/logout          - body: refresh_token; revokes token (204)

  GET    /                     - liveness (no DB); returns { status: 'ok' }
  GET    /health               - readiness (DB check); returns { status: 'ok' } or 503

All request/response bodies validated via Fastify JSON schema. UUIDs in path.


8. TIMER ENGINE (shared logic)
------------------------------
Input: TimerProfile { name, emoji, rounds: [{ name, durationSeconds, warn10sec }] }.
Output: stream of events (callback or advance() per second).

Events:
  round_start  - start of a round (roundIndex, round, totalRounds)
  tick         - every second (roundIndex, round, remainingSeconds, totalRounds)
  warn10       - when remainingSeconds === 10 and warn10sec (roundIndex, round, totalRounds)
  round_end    - end of round
  training_end - all rounds done

Pure logic; no I/O. Used by backend (Telegram) and by Android (Kotlin reimplementation).


9. TELEGRAM BOT
---------------
UI: Button-based (inline keyboards). No need to type commands; user taps buttons.

/start: Welcome message (i18n by user language_code) with two buttons: "Profiles", "Quick timer".
  - Profiles: Lists profiles from API as buttons (emoji + name); tap to start that profile's timer. "Back" returns to main menu.
  - Quick timer: Presets as buttons — e.g. 3×5+1, 5×3+1, "5 run, 5 walk". "Back" returns to main menu.
Commands (still supported): /profiles (same list with buttons), /timer <spec> (with no arg shows main menu; with arg starts timer).
<spec> for /timer: "3x5 1" (template), "5 run, 5 walk" (custom), or profile name/ID from API.

Flow: runs timer-engine in process; every second calls advance(). On round_start and warn10, synthesizes speech via Google TTS and sends voice message. On training_end sends text "Training finished." One running timer per user (stored in memory).

Env: TELEGRAM_BOT_TOKEN required. On VPS, API_URL is set so the bot talks to the local API (127.0.0.1:3001). Optional: GOOGLE_APPLICATION_CREDENTIALS for TTS.


10. ANDROID APP
---------------
Location: android/ (Gradle, Kotlin, Compose).

Screens: profile list, profile/round editor (name, emoji, rounds with name/duration/warn10 and label), timer. The UI uses Material 3 components, dynamic colors, elevated cards, and Material icons (`material-icons-extended`). Profile cards now show quick stats (round count + total duration), and Start is disabled for profiles without rounds. The editor uses Material 3 Switches for toggles and handles capitalization. Training (profile) name is limited to 30 characters; round name is limited to 20 characters (input and save). Emoji field limited to a single emoji (BreakIterator grapheme cluster). Each round has a numbered circle: tap to select/deselect, long-press on the circle or card padding to drag and reorder (haptic feedback, visual lift). Delete button on each round. When rounds are selected, a "Copy × N" action duplicates the block 1–99 times (e.g. one work + one rest copied 12 times for boxing, then delete the last rest). The timer features a large dynamic CircularProgressIndicator (thick stroke, 36dp), tabular numerals for the countdown, and a step indicator chip ("1 / N"). The screen is kept ON (`FLAG_KEEP_SCREEN_ON`) during a running workout. Pause freezes the countdown (button becomes "Пауза" / "Pause"); when paused, two buttons appear: Resume ("Продолжить" / "Resume") to continue and Stop ("Стоп" / "Stop") to end the training. A "Пауза" chip appears next to the step indicator when paused. When training finishes (naturally or via Stop), "Тренировка завершена" is shown and "Restart/Повторить" resets the timer. The profile list top bar includes a language selector chip (dropdown menu with 8 languages: EN, RU, UZ, KK, AZ, TG, TT, ZH); the preference is persisted in SharedPreferences and applied via attachBaseContext override on activity recreate. Profile editor: delete profile shows a confirmation dialog (RU/EN) before removal. All icon buttons have localized contentDescription for accessibility. "Add round" label is "Add round" / "Добавить раунд" (icon + text, no duplicate plus). Data in Room; sync with backend on VPS when online (Retrofit). Sync runs only when user saves a training (create/update/delete profile or save rounds) and when connectivity is re-established after being lost; on connectivity restore a 1-minute debounce ensures sync runs only after 1 minute of stable connection. Sync uses GET /profiles?include=rounds (batch, one request per 100 profiles) and optional updated_since for delta; save rounds uses PUT /profiles/:id/rounds (atomic). Throttle: sync skipped if last sync was < 30 s ago. Auth: TokenStore (EncryptedSharedPreferences when available) holds access token and refresh_token; on 401 OkHttp Authenticator calls POST /auth/refresh via synchronous Call and retries with the new token. Timer: same event contract. TTS for round name and "X через 10 секунд" (Russian when device locale is RU) uses USAGE_ALARM stream to prevent volume ducking. One continuous prolonged tone (1200 ms sine wave via AudioTrack, USAGE_ALARM) at training start (first round) and at end of each non-final round. Training end: TTS "Тренировка [name] завершена" (no tone). Round names > 10 chars are truncated to 10 + "…" on the timer display. Short ticks (150 ms, ToneGenerator TONE_CDMA_PIP, STREAM_ALARM) per second in the last 10 sec when "Предупредить за 10 сек" ON and duration >= 10 sec. App waits for TTS ready before first announcement. Accessibility: content descriptions are localized (EN/RU) and Android 14 back-invoked callback is enabled in manifest.

Performance optimizations: (1) TTS pre-caching uses parallel TextToSpeech instances (up to 3 concurrent) to synthesize missing phrases ~2-3x faster than sequential. (2) Profile save is local-first: Room insert returns instantly (<10ms), API call runs in a background coroutine (fire-and-forget); sync reconciles on next connectivity. (3) OkHttp has explicit timeouts (5s connect, 10s read/write) and a connection pool (5, 30s); an interceptor logs HTTP timing. (4) R8 minification enabled for release builds (isMinifyEnabled=true, isShrinkResources=true) with ProGuard rules for Retrofit, Gson, Room, Parcelize; reduces APK from ~20MB to ~1.5MB. (5) Notification updates are throttled: cached Notification.Builder, startForeground called every 5s, notify() for intermediate ticks. (6) Sync upserts wrapped in database.withTransaction{} for single SQLite commit per page. (7) MediaPlayer instance is reused across voice playbacks (reset instead of create/release). (8) Prolonged tone ShortArray buffer is computed once and cached for reuse across round transitions.

Settings: gear icon on profile list top bar opens a dialog (minimal). When the timer is running, TimerService is started (foreground notification) and a PARTIAL_WAKE_LOCK is held so the timer and audio (TTS, tones) continue with screen off; service stops when the timer ends or the user goes back. This prevents OEM power management (e.g. TECNO Hiber) from freezing the process. TimerService runs in a separate process (:timer). POST_NOTIFICATIONS is declared and requested at runtime on Android 13+; notification channel raund_timer_visible (IMPORTANCE_HIGH) shows round name and countdown.

Package: com.raund.app. API base URL in RaundApplication points to the VPS: http://round.ozzy1986.com:8080/ (for real device / phone). For Android emulator only use http://10.0.2.2:3001/.

Testing on phone: the app is usually already installed. Just launch it via ADB
(see ANDROID_TESTING.txt). Only rebuild the APK if android/ code was changed.
  - Rebuild (only when needed): from project root run "scripts\build-apk.ps1" (or "cd android && .\gradlew.bat assembleDebug"). The script copies the APK to android/Round-debug.apk.

14. QUALITY & SECURITY
-----------------------
Aligned with CORE PRINCIPLES above: security, speed, stability, scalability; DRY, KISS, SOLID; TDD with full test coverage.
- Validate all input (Fastify schemas), parameterized queries only, no secrets in logs, .env not committed.
- CORS: configurable via CORS_ORIGINS env var (comma-separated origins; empty = allow all).
- BOT_SECRET comparison uses crypto.timingSafeEqual (SHA-256 digest) to prevent timing attacks.
- Nginx: HTTP 8080 redirects to HTTPS 443. HSTS, X-Content-Type-Options, X-Frame-Options headers. TLS 1.2+. Upstream keepalive and gzip compression for JSON.
- Graceful shutdown: SIGTERM/SIGINT handlers drain in-flight requests and close DB pool before exit.
- N+1 queries eliminated: getProfileWithRoundsById uses single query with json_agg; round update/delete use JOINed ownership checks (1 query instead of 2-3).
- Refresh token rotation: short-lived access token (15m), refresh token (30d) stored hashed in DB; rotation on /auth/refresh; Android retries on 401 via refresh.
- Redis (optional): when REDIS_URL set, rate limit is shared across PM2 workers; bot token/fileId/TTS caches use Redis with TTL. Required at scale (multi-instance). Rate limit key: user id when Bearer present, else IP; RATE_LIMIT_MAX (default 200), RATE_LIMIT_WINDOW (default 1 minute).
- PgBouncer (optional): deploy/pgbouncer.ini.sample; PG_POOL_MAX per worker (default 10); pool.on('error') and exhaustion warning; optional PG_STMT_TIMEOUT_MS.
- Sentry (optional): SENTRY_DSN for 5xx error reporting; health-check.sh for uptime cron.
- Shared timer core; Vitest with coverage (timer-engine 100%, auth, cache, others high).
- Offline-first and data validation. UI: minimal, big countdown, clear typography.
- Timer tick sounds: ToneGenerator at max volume (100), 200ms TONE_CDMA_PIP on STREAM_ALARM; TTS at volume 1.0f; prolonged alarm tone at 0.55f via AudioTrack.
- Screen-off handling: when the user starts the timer, TimerService is started (foreground notification) and acquires PARTIAL_WAKE_LOCK; both are released when the timer ends or the user navigates back. Timer and audio (TTS, AudioTrack, ToneGenerator) then continue with screen off; OEM power management (e.g. TECNO Hiber) does not freeze the process. All audio ops are wrapped in try-catch so failures don't crash the timer coroutine. Permissions: WAKE_LOCK, FOREGROUND_SERVICE, FOREGROUND_SERVICE_SPECIAL_USE, POST_NOTIFICATIONS (runtime on Android 13+).
- Theme: vibrant red primary (#E53935), teal secondary (#00897B), orange tertiary (#FF9800); bold typography; dynamic colors on Android 12+. Timer ring pulses (scale animation) in warning zone (last 10s). Profile cards use rounded corners (20dp), elevated cards with primaryContainer emoji circles.
- Android QA gate: `assembleDebug` + on-device ADB regression + `:app:lintDebug` (current lint: 0 errors; warnings are dependency update suggestions only).
